## 四、安全性层面

### 4.1 ✅ 良好实践

- [x] JWT Token 使用 HS256，15分钟过期
- [x] Refresh Token 轮换（每次刷新都换新 token）
- [x] 权限变更时 Token 撤销（tokenVersion 机制）
- [x] Refresh Token 存储为 SHA-256 哈希
- [x] 密码使用 bcrypt（cost factor 12）
- [x] 生产环境隐藏堆栈跟踪
- [x] 所有请求体使用 Zod 验证

### 4.2 ⚠️ 需要加强

| 问题 | 优先级 | 建议 |
|------|--------|------|
| `revokeSessionsByRoleId` 未清除 tokenVersionCache | 🟡 中 | 受影响用户 60 秒内旧 token 仍有效 |
| 无按用户限流（仅按 IP） | 🟢 低 | 分布式攻击可绕过 |
| `/identity/refresh` 无独立限流 | 🟡 中 | 添加 10次/分钟限制 |
| JwtAuthGuard 非全局守卫 | 🟢 低 | 新 Controller 可能遗漏 |
| `JWT_REFRESH_SECRET` 环境变量未使用 | 🟢 低 | 死配置，易误导 |
| 过期 refresh_sessions 无自动清理 | 🟢 低 | 添加定时清理任务 |
